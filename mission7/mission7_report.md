# 📊 똑닥 서비스 AARRR 기반 지표 설계 

## 1. 서비스 개요

똑닥은 병원 예약 및 접수 서비스를 제공하는 모바일 플랫폼이다.
사용자는 병원 검색, 예약, 대기 현황 확인 등을 통해 병원 이용 과정을 효율적으로 관리할 수 있다.

최근에는 연간 구독형 멤버십(유료 서비스)을 통해 예약 편의성 및 추가 혜택을 제공하며 수익 구조를 확장하고 있다.

본 보고서는 똑닥 서비스의 사용자 흐름을 기반으로 AARRR 프레임워크를 적용하여 핵심 지표를 설계하고
각 지표의 목적, 측정 방식 그리고 데이터 기반 활용 방안을 제시하는 것을 목표로 한다.

---

## 2. AARRR 프레임워크 정의

| 단계          | 정의                |
| ----------- | ----------------- |
| Acquisition | 앱 유입 및 회원가입       |
| Activation  | 병원 검색 및 첫 예약 완료   |
| Retention   | 재방문 및 재예약         |
| Revenue     | 예약 및 멤버십 기반 수익 발생 |
| Referral    | 추천 및 자연 유입        |

---

## 3. 단계별 주요 지표 설계

---

### 3.1 Acquisition (유입)


| 지표             | 목적             | 측정 방식              |
| -------------- | -------------- | ------------------ |
| 앱 설치 수         | 전체 유입 규모 파악    | 앱 스토어 다운로드 수       |
| 회원가입 전환율       | 유입 대비 가입 효율 분석 | 회원가입 수 / 앱 방문자 수   |
| 병원 검색 진입률      | 서비스 관심도 파악     | 병원 검색 사용자 / 전체 방문자 |

---

### 3.2 Activation (활성화)


| 지표               | 목적               | 측정 방식                 |
| ---------------- | ---------------- | --------------------- |
| 첫 예약 완료율         | 핵심 기능 경험 여부 판단   | 첫 예약 완료 유저 / 전체 가입자   |
| 예약 전환율           | 검색 → 예약 전환 효율 분석 | 예약 완료 수 / 병원 검색 사용자 수 |
| 예약 소요 시간         | UX 편의성 평가        | 검색부터 예약까지 평균 시간       |
| 기능 사용률 (대기 확인 등) | 핵심 기능 활용도 측정     | 기능 이용 유저 / 전체 유저      |

---

### 3.3 Retention (재방문)


| 지표            | 목적                | 측정 방식           |
| ------------- | ----------------- | --------------- |
| 7일 / 30일 재방문율 | 서비스 지속 사용 여부 판단   | 기간 내 재접속 유저 비율  |
| 재예약률          | 반복 사용 패턴 분석       | 재예약 유저 / 전체 유저  |
| 진료과목별 재방문율    | 질병/목적 기반 이용 패턴 분석 | 특정 진료과 재이용 비율   |
| 평균 이용 주기      | 이용 간격 분석          | 사용자별 예약 간 평균 기간 |

---

### 3.4 Revenue (수익)


| 지표               | 목적             | 측정 방식              |
| ---------------- | -------------- | ------------------ |
| 멤버십 가입률          | 유료 전환율 분석      | 멤버십 가입자 / 전체 사용자   |
| 멤버십 유지율          | 지속 결제 여부 확인    | 갱신 사용자 / 멤버십 사용자   |
| 멤버십 ARPU         | 유료 사용자 수익성 분석  | 멤버십 매출 / 멤버십 사용자 수 |

---

### 3.5 Referral (추천)

현재 똑닥 서비스는 명확한 추천(Referral) 기능이 활성화되어 있지 않다.
따라서 향후 서비스 확장을 위해 추천 기반 유입 구조를 설계할 필요가 있다.

| 지표          | 목적           | 측정 방식               |
| ----------- | ------------ | ------------------- |
| 추천 가입 비율    | 추천 기반 유입 분석  | 추천 가입자 / 전체 가입자     |
| 추천 → 예약 전환율 | 추천 유저의 질 평가  | 추천 유저 예약 수 / 추천 가입자 |
| 바이럴 계수      | 서비스 확산력 측정   | 초대 수 × 가입 전환율       |
| 공유 클릭 수     | 추천 행동 활성도 측정 | 공유 버튼 클릭 수          |
