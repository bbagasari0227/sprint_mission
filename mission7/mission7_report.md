# 📊 똑닥 서비스 AARRR 기반 지표 설계

## 1. 서비스 개요

*똑닥*은 병원 예약 및 접수 서비스를 제공하는 모바일 플랫폼이다.
사용자는 병원 검색, 예약, 대기 현황 확인 등을 통해 병원 이용 과정을 효율적으로 관리할 수 있다.

본 보고서는 똑닥 서비스의 사용자 흐름을 기반으로 AARRR 프레임워크를 적용하여 핵심 지표를 설계하고,
각 지표의 목적과 측정 방식, 그리고 데이터 기반 활용 방안을 제시하는 것을 목표로 한다.

---

## 2. AARRR 프레임워크 정의

| 단계          | 정의               |
| ----------- | ---------------- |
| Acquisition | 앱 유입 및 회원가입      |
| Activation  | 병원 검색 및 첫 예약 완료  |
| Retention   | 재방문 및 재예약        |
| Revenue     | 예약 및 제휴 기반 수익 발생 |
| Referral    | 추천 및 자연 유입       |

---

## 3. 단계별 주요 지표 설계

---

### 3.1 Acquisition (유입)

⭐ **핵심 지표: 회원가입 전환율**

| 지표             | 목적             | 측정 방식              |
| -------------- | -------------- | ------------------ |
| 앱 설치 수         | 전체 유입 규모 파악    | 앱 스토어 다운로드 수       |
| 회원가입 전환율       | 유입 대비 가입 효율 분석 | 회원가입 수 / 앱 방문자 수   |
| 병원 검색 진입률      | 서비스 관심도 파악     | 병원 검색 사용자 / 전체 방문자 |
| CAC (유저 획득 비용) | 마케팅 효율성 분석     | 마케팅 비용 / 신규 가입자 수  |

**📌 활용 방안**

* 회원가입 전환율이 낮다면 가입 절차 간소화 및 UX 개선 필요
* 특정 채널의 CAC가 높다면 마케팅 채널 재조정 필요

---

### 3.2 Activation (활성화)

⭐ **핵심 지표: 첫 예약 완료율**

| 지표               | 목적               | 측정 방식                 |
| ---------------- | ---------------- | --------------------- |
| 첫 예약 완료율         | 핵심 기능 경험 여부 판단   | 첫 예약 완료 유저 / 전체 가입자   |
| 예약 전환율           | 검색 → 예약 전환 효율 분석 | 예약 완료 수 / 병원 검색 사용자 수 |
| 예약 소요 시간         | UX 편의성 평가        | 검색부터 예약까지 평균 시간       |
| 기능 사용률 (대기 확인 등) | 핵심 기능 활용도 측정     | 기능 이용 유저 / 전체 유저      |

**📌 활용 방안**

* 예약 전환율이 낮다면 병원 검색 UI 및 정보 제공 방식 개선 필요
* 예약 소요 시간이 길다면 입력 과정 단순화 및 UX 개선 필요

---

### 3.3 Retention (재방문)

⭐ **핵심 지표: 재예약률**

| 지표            | 목적                | 측정 방식           |
| ------------- | ----------------- | --------------- |
| 7일 / 30일 재방문율 | 서비스 지속 사용 여부 판단   | 기간 내 재접속 유저 비율  |
| 재예약률          | 반복 사용 패턴 분석       | 재예약 유저 / 전체 유저  |
| 진료과목별 재방문율    | 질병/목적 기반 이용 패턴 분석 | 특정 진료과 재이용 비율   |
| 평균 이용 주기      | 이용 간격 분석          | 사용자별 예약 간 평균 기간 |

**📌 활용 방안**

* 재예약률이 낮다면 리마인드 알림 및 재방문 유도 기능 강화 필요
* 특정 진료과 재방문율이 낮다면 해당 카테고리 UX 개선 필요

---

### 3.4 Revenue (수익)

⭐ **핵심 지표: LTV (고객 생애 가치)**

| 지표               | 목적             | 측정 방식            |
| ---------------- | -------------- | ---------------- |
| ARPU (유저당 평균 수익) | 사용자당 수익성 분석    | 총 매출 / 전체 사용자 수  |
| 예약당 평균 수익        | 예약 기반 수익 구조 분석 | 총 매출 / 총 예약 수    |
| LTV (고객 생애 가치)   | 장기 수익성 평가      | 평균 수익 × 평균 이용 기간 |
| 제휴 병원당 매출        | 병원별 수익 구조 분석   | 병원별 발생 매출        |

**📌 활용 방안**

* LTV가 낮다면 단기 이용 후 이탈하는 구조 → 장기 이용 유도 전략 필요
* 병원별 매출 차이가 크다면 제휴 전략 및 병원 추천 로직 개선 필요

---

### 3.5 Referral (추천)

⭐ **핵심 지표: 초대 코드 사용률**

| 지표          | 목적           | 측정 방식               |
| ----------- | ------------ | ------------------- |
| 초대 코드 사용률   | 추천 기반 유입 분석  | 추천 가입자 / 전체 가입자     |
| 바이럴 계수      | 서비스 확산력 측정   | 초대 수 × 가입 전환율       |
| 공유 클릭 수     | 추천 행동 활성도 측정 | 공유 버튼 클릭 수          |
| 추천 → 예약 전환율 | 추천 유저의 질 평가  | 추천 유저 예약 수 / 추천 가입자 |

**📌 활용 방안**

* 추천 대비 예약 전환율이 낮다면 유입 유저의 질이 낮을 가능성 존재
* 공유 클릭 수가 낮다면 추천 UX 및 인센티브 개선 필요

---

## 4. 결론

똑닥 서비스의 핵심 성과는 “예약 전환”과 “재예약”에 의해 결정된다.

Activation 단계에서 첫 예약 완료율이 낮다면 서비스의 핵심 가치가 충분히 전달되지 않은 것이며,
Retention 단계에서 재예약률이 낮다면 사용자가 일회성 이용에 그치고 있음을 의미한다.

따라서 똑닥 서비스는

1. 병원 검색 및 예약 UX 개선
2. 재방문 유도를 위한 알림 및 리마인드 기능 강화
   를 중심으로 개선 전략을 수립해야 한다.

이와 같은 AARRR 기반 지표 설계를 통해 사용자 흐름을 체계적으로 분석하고,
데이터 기반 의사결정을 수행할 수 있다.
